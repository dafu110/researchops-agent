# ResearchOps Agent 闭环就绪报告

## 结论

本地隔离运行已验证为一个完整的“可控、可追溯、可恢复”的企业研究 Agent 闭环。结论基于一键自检、API 生命周期回归和 32 条评测门禁，而不是仅基于界面演示。

## 已验证的控制闭环

| 阶段 | 触发与状态 | 可观察证据 | 退出或恢复路径 |
| --- | --- | --- | --- |
| 资料范围与研究请求 | `AskRequest` 指定授权资料范围 | 引用、`FinalAnswer`、`TraceStep.input_payload` | 资料不足时返回受限证据提示 |
| 规划 | `PlanStepDetail` 生成阶段、风险与审批需求 | `/api/runs/{id}/trace` | 固定的受限工具调用数量 |
| 工具执行 | `ToolCallRecord` 记录参数、超时、尝试、幂等键 | `/api/runs/{id}/tools`、审计记录 | 完成、失败、超时、阻止、取消、重试状态 |
| 风险控制 | Planner 与工具策略双重审批 | 审批记录与轨迹 | 管理员批准后显式恢复同一 `run_id` |
| 失败与取消 | 工具边界协作式取消；超时可恢复 | 取消/失败轨迹、工具状态 | `/cancel`、`/recover`，恢复复用原输入和幂等键 |
| 最终结论 | `FinalAnswer` 包含引用、模型、Token 与成本字段 | 回答、引用、运行详情 | 无引用时明确标识证据不足 |
| 历史治理 | 管理员删除研究内容 | 删除审计；删除后 trace/replay 为 `404` | 内容最小化的删除留痕 |
| 质量与运行可见性 | 评测、指标和 JSON Schema | `/metrics`、`/contracts`、评测结果 | 指标用于发现失败、审批与成本覆盖异常 |

## 自检结果

运行：

```powershell
python scripts\run_agent_self_check.py
```

本次自检通过以下七项：带引用研究与轨迹、工具生命周期、审批后的同一运行恢复、协作式取消、超时后的失败恢复、契约/指标/删除治理，以及 32 条评测门禁。自检发现并修复了一个指标口径缺陷：已恢复运行不再仅依赖当前 `requires_approval` 状态计算审批率，而是从持久化审批记录按运行去重计算。

## 界面映射

界面保持“研究、资料、治理”的分层：

- **研究**：结论、引用、计划与闭环状态。
- **治理 → 运行指标**：成功率、P95、工具失败率、审批率与成本覆盖。
- **运行详情**：每步输入/输出、模型、Token、耗时；工具参数、超时、尝试、幂等键；取消与失败恢复入口。

## 未验证边界

以下项目被明确排除在“本地闭环已验证”的结论外：

- PostgreSQL 多实例并发、迁移回滚、备份和灾难恢复。
- 真实第三方 MCP、写入型连接器、凭证轮换和外部幂等语义。
- 生产真实模型的精确 Token 账单、成本预算拦截与限流。
- Docker sandbox 逃逸、资源争抢和长时间运行的取消时效。
- 真实业务数据、人工标注和领域专家的离线质量校准。

这些边界已在 README 与控制规范中标注，不应被包装为已完成的生产能力。
