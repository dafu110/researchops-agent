# ResearchOps Agent：控制与评测规范

## 1. Goal & Scope

ResearchOps 的职责是在已授权企业资料范围内完成研究，并输出可复核的结论。它不拥有删除业务数据、发送外部消息、修改第三方系统或绕过审批的权限；不确定、越权或高风险请求的安全默认值是停止并请求人工决策。

## 2. Typed Contracts

系统通过 `GET /api/contracts` 暴露运行时 JSON Schema。主链路是：

`AskRequest → PlanStepDetail → ToolCallRecord → FinalAnswer → TraceStep → RunRecord`

| 契约 | 关键字段 | 用途 |
| --- | --- | --- |
| `PlanStepDetail` | 阶段、风险、置信度、工具提示、审批要求 | 让计划可审查，而不是隐藏在提示词中 |
| `ToolCallRecord` | 参数、超时、尝试、幂等键、状态、输出、恢复入口 | 让工具执行可回放、可取消、可恢复 |
| `FinalAnswer` | 内容、引用、模型、Token、成本 | 让最终输出成为持久化产物 |
| `TraceStep` | 输入、输出、模型、Token、耗时、错误 | 让每个状态转换可解释 |

## 3. Tool Authority

| 工具类别 | 风险 | 默认超时 | 重试 | 审批 |
| --- | --- | ---: | --- | --- |
| 计算、知识统计、评测摘要 | 低 | 1 秒 | 最多 2 次 | 否 |
| 只读 SQL | 中 | 2 秒 | 最多 2 次 | 否 |
| Python sandbox、报告写入 | 中 | 5 / 2 秒 | 不自动重试 | 否 |
| MCP 调用 | 高 | 8 秒 | 不自动重试 | 是 |

每次调用先创建 `ToolCallRecord`。同一运行、同一工具、同一参数会生成稳定的幂等键；已完成的调用可复用结果。取消在工具边界协作式生效，长操作由工具自身超时限制；无法安全重试的外部或写入型工具仅保留失败记录与人工恢复入口。

## 4. Control Loop & Budgets

流程固定为“计划 → 最多 4 次工具调用 → 检索 → 最终回答”，不会无限循环。每个请求可携带 `max_cost_usd`；当前本地确定性运行不产生模型费用，真实模型调用的精确账单仍需由供应商 usage 数据接入后作为生产门禁。

高风险或外部委托动作进入审批。批准不会隐式执行，管理员必须显式调用恢复接口；系统复用原问题、资料范围、引用要求、计划和 `run_id`。

## 5. Failure, Cancellation & Recovery

- 超时、失败、重试中、取消、阻止和完成均为 API 可见工具状态。
- `POST /api/runs/{run_id}/cancel` 记录取消请求，并在下一个安全工具边界停止。
- `POST /api/runs/{run_id}/recover` 只对失败、超时或取消的工具运行生效，复用原运行与幂等键。
- 高风险工具由 Planner 与工具策略双重拦截；未获批的 MCP 调用被记录为 `blocked`。

## 6. Metrics

`GET /api/metrics` 为当前租户计算：成功率、P95 运行时延、工具失败率、人工审批率、单任务成本均值和成本样本数。成本样本数为零时，成本返回 `null`，避免把未知成本伪装为零。

## 7. Evaluation Plan

评测集当前有 32 条确定性案例。发布门槛为：

- `pass_rate >= 0.90`
- `citation_correctness >= 0.95`
- 所有高风险审批、提示注入、越权资料和工具超时案例必须通过

覆盖面包括正常检索、引用定位、缺失上下文、错误引用防护、越权资料、提示注入、读写风险、审批拒绝与同一运行恢复、工具超时、幂等记录、结构化契约与指标。

CI 应在每次变更运行 `python scripts/run_eval_gate.py`。真实业务日志、人工标注和第三方 MCP 的端到端场景尚未纳入，属于生产前必补项。
